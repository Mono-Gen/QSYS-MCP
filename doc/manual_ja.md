# Q-SYS MCP サーバー 導入・機能マニュアル (v0.9.0)

本ドキュメントは、Q-SYS MCP サーバー (`qsys-mcp`) v0.9.0 を導入して起動するための手順、および提供される全機能（MCPツール）の使い方をまとめた、**初心者向けのガイドブック**です。
プログラミングやデザインに慣れていない方でも、AIアシスタントと協力しながらQ-SYSの制御や画面の着せ替え（CSS）ができるよう、分かりやすく解説しています。

---

## 🔰 はじめに：基本コンセプトと用語集

本サーバーやAIを使って開発を始める前に、まずは基本となる言葉を整理しましょう！

*   **Q-SYS（キューシス）**：音響、映像、そして様々な機器の制御をまとめて行うためのシステムです。
*   **Core（コア）**：Q-SYSシステムの「頭脳」にあたる機器です。すべての音響処理や制御プログラムがここで動きます。
*   **MCP（Model Context Protocol）**：AI（ClaudeやAntigravityなど）が、あなたのパソコンやQ-SYS Coreと直接通信して、設定を読み取ったりプログラムを書き換えたりするための仕組みです。
*   **Named Control（名前付きコントロール）**：外部（AIや他の機器）から操作できるように、名前をつけたボタンやボリュームのことです。テレビのリモコンのボタンのようなイメージです。
*   **Component（コンポーネント）**：Q-SYS Designer（設計ソフト）に配置する、ミキサーやカメラ、制御用スクリプトなどの「機能ブロック」のことです。
*   **UCI（ユーザー・コントロール・インターフェース）**：iPadや専用タッチパネルに表示する、ユーザーが操作するための「カスタム画面」のことです。
*   **Lua（ルア）**：Q-SYSの中で「ボタンが押されたら、アンプの電源を入れる」といった自動化ルールを書くための、軽くてシンプルなプログラミング言語です。
*   **CSS（シーエスエス）**：UCI（操作画面）のボタンの色、文字の大きさ、背景の見た目をオシャレに飾り付けるための「デザインシート」です。

## 導入と起動の手順

### 1. 配布ファイルの構成
実行に必要なファイルは以下の2点です。これらを同一のフォルダに配置してください。
*   `qsys-mcp.exe`（またはお使いのOS用のバイナリ）
*   `config.json`（設定ファイル）

### 2. 設定ファイル (`config.json`) の編集
お使いの Q-SYS Core のネットワーク環境や接続情報に合わせて、`config.json` をメモ帳などのテキストエディタで編集します。

*   **接続先の Core にパスワード（Access Control）がかかっている場合**
    `cores` の値に以下のようにユーザー名とPIN（パスワード）を追記します。
    *   **書式**: `"cores": "エイリアス=IPアドレス:QRCポート:ECPポート:ユーザー名:パスワード(PIN)"`
    *   **設定例**:
        ```json
        {
          "cores": "default=127.0.0.1:1710:1702:admin:1234",
          "styles_dir": "<Q-SYS DesignerのStylesフォルダの絶対パス（例: Windowsのドキュメントフォルダ配下など）>",
          "lua_reference_path": "reference/reference_lua.md",
          "css_reference_path": "reference/reference_css.md",
          "polling_interval": 350,
          "debug": true
        }
        ```
*   **パスワードがかかっていない場合**
    ポート番号や認証情報は省略可能です。
    *   **設定例**: `"cores": "default=127.0.0.1"`

### 設定パラメータの詳細 (`config.json` オプション)

*   **`cores`** (string, 必須)
    *   **意味**: 接続先となる Q-SYS Core の接続情報。
    *   `エイリアス`: Coreを識別するための任意の名前（例: `default`、`sub-core`）。
    *   `IPアドレス`: Q-SYS Core のネットワーク上のIP（またはホスト名）。
    *   `QRCポート`: Q-SYS Remote Control プロトコルのポート（通常は `1710` 固定）。
    *   `ECPポート`: External Control Protocol のポート（通常は `1702` 固定）。
    *   `ユーザー名` / `パスワード(PIN)`: Q-SYS Coreの認証機能が有効な場合に入力する管理者アカウント情報。
*   **`styles_dir`** (string, 必須)
    *   **意味**: Q-SYS Designer のスタイルシート（CSS）ファイルを保存するためのフォルダパス。
    *   **詳細**: Windowsの場合、通常は `C:\Users\<ユーザー名>\Documents\QSC\Q-SYS Designer\Styles` のようなフォルダを指定します。このフォルダ配下にあるCSSファイルを、AIが直接編集・閲覧できるようになります。
*   **`lua_reference_path`** (string, 必須)
    *   **意味**: Q-SYS Lua API 仕様書（Markdown形式）のファイルパス。
    *   **詳細**: デフォルトでは `reference/reference_lua.md` が設定されています。AIアシスタントがLuaの書き方を調べる際に読み込みます。
*   **`css_reference_path`** (string, 必須)
    *   **意味**: Q-SYS UCI CSS スタイリング仕様書（Markdown形式）のファイルパス。
    *   **詳細**: デフォルトでは `reference/reference_css.md` が設定されています。AIアシスタントがCSSの書き方を調べる際に読み込みます。
*   **`polling_interval`** (number, 任意)
    *   **意味**: Q-SYS Core への状況確認（ポーリング）の間隔（ミリ秒）。デフォルト値は `350`。
*   **`debug`** (boolean, 任意)
    *   **意味**: 開発者向けのデバッグログを出力するかどうか。デフォルト値は `false`。
*   **`protected_controls`** (string, 任意)
    *   **意味**: 外部からの書き換えを制限（プロテクト）するコントロール名を指定します。
    *   **詳細**: 例として `"system\\..*,.*\\.password"` と記述すると、`system.` で始まるコントロールや、`.password` で終わるコントロールの値を外部から誤って変更（セット）しようとした際に、書き込みをブロックし安全性を保ちます。なお、コンポーネント内のコントロールに対しては `コンポーネント名.コントロール名`（例: `ImportantMic.mute`）の形式で判定が行われます。

### 3. 各OSでの実行方法

#### ■ Windows の場合
1. `qsys-mcp.exe` と `config.json` を同じフォルダに置きます。
2. コマンドプロンプトやPowerShellから実行します。
   ```powershell
   .\qsys-mcp.exe
   ```

#### ■ Mac の場合 (Apple Silicon/Intel)
1. ご自身のMacのCPUタイプ（M1/M2/M3等は `arm64`、Intel製CPUは `amd64`）に対応するバイナリファイルを用意し、`qsys-mcp` にリネームします。
2. `config.json` と同じフォルダに置きます。
3. **実行権限の付与 (初回のみ)**:
   ターミナルを開き、配置したフォルダに移動して以下のコマンドを実行してプログラムに実行権限を与えます。
   ```bash
   chmod +x ./qsys-mcp
   ```
4. **開発元未確認警告の回避 (初回のみ)**:
   Finderでバイナリファイルを **右クリック（または Ctrl を押しながらクリック）して「開く」** を選択します。警告画面が出ますが、「開く」を選択することでMacに実行を許可させることができます。
5. ターミナルから起動します。

### 4. 各種AIアシスタント (Claude, Antigravity等) との連携

本サーバーをAIアシスタントに登録することで、AIがあなたの指示に従って自動的にQ-SYSの情報を取得したり、ファイルを書き換えたりできるようになります。

#### 1. 設定ファイルの場所
お使いのツールとOSに合わせて、以下の設定ファイルをテキストエディタで編集します。

*   **Claude Desktop**
    *   **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
    *   **Mac**: `~/Library/Application Support/Claude/claude_desktop_config.json`
*   **Claude Code (CUI版)**
    *   **共通**: `~/.claude.json`
*   **Antigravity 2.0 / MCP対応AIエージェント**
    *   **共通**: `~/.gemini/antigravity/mcp_settings.json`

#### 2. 設定ファイルの記述例

##### ■ Windows の場合
Windows環境では、以下のように `cmd.exe /c` を経由して起動させることで、安全かつ確実に動作させることができます。

```json
{
  "mcpServers": {
    "qsys-mcp": {
      "command": "cmd.exe",
      "args": [
        "/c",
        "<qsys-mcp.exeが配置されているフォルダの絶対パス>\\qsys-mcp.exe"
      ],
      "cwd": "<config.jsonが配置されているフォルダの絶対パス>"
    }
  }
}
```
※ パス区切り文字は、必ず **バックスラッシュ（円マーク）を2つ重ねた `\\`** で記述してください。

##### ■ Mac の場合
Macでは、バイナリファイルを直接指定して動作させることができます。

```json
{
  "mcpServers": {
    "qsys-mcp": {
      "command": "<qsys-mcpが配置されているフォルダの絶対パス>/qsys-mcp",
      "args": [],
      "cwd": "<config.jsonが配置されているフォルダの絶対パス>"
    }
  }
}
```

##### ■ 開発者向け：Go で直接実行する場合（Windows / Mac 共通）
Go SDKがインストールされている環境であれば、実行ファイルをビルドすることなく、直接ソースコードから起動させることができます。コードの変更が即座に反映されるため、開発作業時におすすめです。

```json
{
  "mcpServers": {
    "qsys-mcp-dev": {
      "command": "go",
      "args": [
        "run",
        "."
      ],
      "cwd": "<プロジェクトのルートフォルダの絶対パス>"
    }
  }
}
```
※ `cwd` には、`go.mod` や `main.go` が配置されているプロジェクトのルートディレクトリの絶対パスを指定してください。

## 提供される機能（MCPツール）一覧

AIアシスタントは、以下のツールを必要に応じて自動的に使い分けます。手動で呼び出す必要はありませんが、どのようなツールがあるか知っておくと指示がしやすくなります。

### 1. 接続管理 (Connection)
*   **`qsys_list_cores`**
    *   **何をする？**: 接続できる Q-SYS Core の一覧を取得します。
*   **`qsys_core_status`**
    *   **何をする？**: 指定した Core が現在動いているか、デザイン名（設定ファイル）は何かを取得します。

### 2. コントロール制御 (Controls)
画面上のボリュームやボタンなど、「名前」が付いたコントロールを操作します。
*   **`qsys_get_control`**
    *   **何をする？**: 指定した名前のボタンが「ON」か「OFF」か、ボリュームが「何dB」かなどの状態を調べます。
*   **`qsys_set_control`**
    *   **何をする？**: 指定した名前のボタンを押したり、ボリュームの値を変更したりします。
*   **`qsys_get_controls`**
    *   **何をする？**: 複数のコントロールの状態をまとめて一括で調べます。

### 3. コンポーネント制御 (Components)
Q-SYS内の各種機器ブロック（ミキサーなど）の中身を操作します。
*   **`qsys_list_components`**
    *   **何をする？**: Q-SYSの中に配置されているすべての機器ブロックの名前一覧を取得します。
*   **`qsys_get_component_controls`**
    *   **何をする？**: 特定のミキサーなどの内部にあるすべてのつまみやボタンの状態を取得します。
*   **`qsys_set_component_controls`**
    *   **何をする？**: 特定のミキサーなどのつまみやボタンを一括で変更します。

### 4. スナップショット制御 (Snapshots)
お気に入りの音量や設定状態を「保存」したり「呼び出し」たりします。
*   **`qsys_list_snapshots`**
    *   **何をする？**: 保存されているスナップショット（設定パターン）の一覧を取得します。
*   **`qsys_load_snapshot`**
    *   **何をする？**: 保存された設定パターンを呼び出します（何秒かけて音量を変化させるかも指定できます）。
*   **`qsys_save_snapshot`**
    *   **何をする？**: 現在の状態をスナップショットに保存します。

### 5. チェンジグループ (Change Groups)
*   **`qsys_create_change_group` / `qsys_poll_change_group` / `qsys_destroy_change_group`**
    *   **何をする？**: 複数のボタンや音量を見張り、**「値が変わったときだけ」** AIに通知するための効率化機能です。

### 6. UCI 制御 (UCI)
iPadなどの操作画面（UCI）を遠隔でコントロールします。
*   **`qsys_set_uci_page`**
    *   **何をする？**: iPadの画面を「音量調整ページ」や「プロジェクター操作ページ」へ強制的に切り替えます。
*   **`qsys_get_uci_status`**
    *   **何をする？**: 現在iPadなどの画面にどのページが表示されているかを調べます。

### 7. ローカルCSS操作 (Local CSS)
*   **`qsys_write_local_css` / `qsys_read_local_css`**
    *   **何をする？**: パソコンの中にある Q-SYS 画面用のデザインシート（CSS）を読み書きします。

### 8. ローカルリファレンス参照 (Local Reference)
*   **`qsys_get_css_reference` / `qsys_get_lua_reference`**
    *   **何をする？**: AIがQ-SYS特有のプログラムの書き方や、デザインの書き方を忘れたときに、説明書を読み取ってカンニングするための機能です。

## AIを活用した開発手順（初心者向け）

AIアシスタントをパートナーにして、Q-SYSの「プログラム（Lua）」や「画面デザイン（CSS）」を楽しく作成していきましょう！

---

### 1. Luaスクリプトを記述する場合

Q-SYSには、自動制御や外部機器との通信を行うために「Lua（ルア）」というプログラミング言語が用意されています。

#### 💡 初心者が最もハマる「お掃除ロボット（GC）の罠」
Luaスクリプトを書く上で、初心者が必ずぶつかる大きな壁があります。それが **「ガーベジコレクション (GC)」** という仕組みです。

*   **何が起きる？**
    Luaには、使わなくなったメモリを自動で掃除する「お掃除ロボット（GC）」が裏で動いています。
    プログラムの中で一時的に使う変数（頭に `local` をつけたもの）で通信用のソケットやタイマーを作ると、プログラムの処理が一段落した瞬間に、お掃除ロボットが**「あ、これはもう使い終わったゴミだな」と勘違いして、勝手に消去（回収）してしまいます。** その結果、通信が突然切れたり、タイマーが動かなくなったりします。

*   **どうやって防ぐ？**
    お掃除ロボットに片付けられないようにするために、**「ずっと使い続ける大事なものだよ」** と明示する必要があります。
    
    1.  **対策A：`local` をつけない（グローバル変数にする）**
        ```lua
        -- ❌ ダメな例：お掃除ロボットに片付けられて接続が切れる
        local myTimer = Timer.New()
        
        -- ⭕ 良い例：ずっと残るため、正常に動き続ける
        myTimer = Timer.New()
        ```
    2.  **対策B：スクリプトの最上部（一番上）で宣言する**
        ファイルの最上部で `local` 変数として宣言しておけば、スクリプト全体が生きている間は回収されません。
        ```lua
        -- ⭕ 良い例：ファイルの先頭で定義しておく
        local socket = TcpSocket.New()
        
        function ConnectDevice()
            socket:Connect("192.168.1.10", 80)
        end
        ```

#### 🛠️ AIを使ったLua開発のステップ
1.  **AIに「Q-SYS専用のルール」を読み込ませる**
    チャットの最初に、以下のプロンプトを送ってAIにルールを確認させます。
    > **プロンプトの例**：
    > 「Q-SYSで動かすためのLuaスクリプトを作成したいです。まず `qsys_get_lua_reference` を呼び出して、通信オブジェクト（TcpSocketなど）がお掃除ロボット（GC）に消されないようにするための書き方や、Q-SYS固有のルールを確認してください。」
2.  **作りたい機能を指示する**
    > **プロンプトの例**：
    > 「プロジェクターの電源をON/OFFするスクリプトを作りたいです。IPアドレス `192.168.1.100`、ポート `23` に対して、ONのときは `'PWR ON\r'`、OFFのときは `'PWR OFF\r'` というTCPコマンドを送るコードを作成してください。お掃除ロボット対策も忘れずにお願いします。」
3.  **動作確認と適用**
    AIが作成したコードを、Q-SYS Designerの「Text Controller」などのスクリプト入力欄にコピー＆ペーストして実行し、動作テストを行います。

---

### 2. UCI CSSスタイルシートを記述する場合

Q-SYSの操作画面（UCI）は、CSSというデザインシートを使うことで、色や形、アニメーションなどを自由に着せ替えできます。

#### 💡 CSSデザインが画面に反映されるまでの全体像
「CSSを書いても、どうやって実際の画面に適用すればいいか分からない！」という場合は、以下の4ステップを順番に行います。

```
【ステップ1】AIがCSSを書く（qsys_write_local_css を使用）
       ↓
【ステップ2】Q-SYS Designerの「Styles」フォルダにCSSファイルが保存される
       ↓
【ステップ3】Designerの画面（UCI）のプロパティで、そのCSSファイルを登録する
       ↓
【ステップ4】画面上のボタンなどのプロパティの「Class」に、CSSで決めた名前を入力する
```

#### 🛠️ AIを使ったCSS開発のステップ

1.  **AIに「Q-SYS UCI専用のスタイルルール」を読み込ませる**
    まずは、Q-SYSの画面装飾で使える特別な名前やプロパティのルールをAIに把握させます。
    > **プロンプトの例**：
    > 「Q-SYSの操作画面をCSSできれいにしたいです。まず `qsys_get_css_reference` を呼び出して、UCIで使える特別なボタンの状態（アクティブ時など）やフォントのルールを読み込んでください。」
2.  **デザインの要望を伝える**
    AIに作りたいデザインのイメージを伝えます。
    > **プロンプトの例**：
    > 「マイクのミュートボタン（クラス名: `mute-btn`）のデザインを作成してください。ミュートがONの時は赤色で、うっすら点滅するようなアニメーションをつけ、OFFの時は落ち着いた灰色になるようなCSSを書いて、`custom_style.css` という名前で保存してください。」
3.  **Q-SYS Designer側での設定（仕上げ）**
    AIが `qsys_write_local_css` を使ってファイルを保存したら、Q-SYS Designerを開きます。
    *   **手順A**: UCIの背景など、何も配置されていない場所をクリックします。
    *   **手順B**: 右側のプロパティパネルにある「Style Sheet」という項目で、AIが書き出したCSSファイル名（例: `custom_style.css`）を選択します。
    *   **手順C**: 装飾したいマイクボタンを選択し、プロパティパネルの「CSS Class」欄に、AIが決めたクラス名である `mute-btn` を入力します。これでボタンのデザインがリアルタイムに変化します！

#### 🎨 初心者向け：よく使う簡単なCSSデザイン例
AIに「こういうコードを書いて」と頼むための参考例です。

*   **ボタンに触れたとき（ホバー）に少し浮かび上がらせる**
    ```css
    /* ボタンにカーソルを乗せたとき、少し明るくして影をつける */
    .btn-hover-effect:hover {
      opacity: 0.9;
      box-shadow: 0px 4px 10px rgba(0,0,0,0.3);
    }
    ```
*   **緊急停止ボタンなどをピカピカ点滅させる**
    ```css
    /* 赤い点滅アニメーション */
    .alert-flash {
      background-color: #FF0000;
      animation: blinker 1.5s linear infinite;
    }
    @keyframes blinker {
      50% { opacity: 0.3; }
    }
    ```

## 困ったときは？（トラブルシューティング）

*   **Q. AIが「ファイルを保存できない」とエラーを出します**
    *   **A.** `config.json` 内の `styles_dir` に指定したフォルダのパスが間違っている可能性があります。実在するフォルダパスが正しく指定されているか確認してください。
*   **Q. Luaスクリプトが一度動いてから、すぐに動かなくなってしまいます**
    *   **A.** 「お掃除ロボット（GC）の罠」にかかっている可能性が高いです。プログラムの作成時、AIに「お掃除ロボット対策のグローバル変数になっているか確認して」と指示し、コードを修正してもらってください。
*   **Q. CSSのクラス名をボタンに入力したのに、デザインが変わりません**
